📐 Mathematics for Data Science
DomainKey TopicsWhy It Matters
Linear Algebra Vectors, matrices, matrix multiplication, transpose, inverse, determinant, eigenvalues & eigenvectors, SVD, dot product, projections Core of ML (weights = matrices). PCA, neural networks, embeddings all use LA.
Probability Sample space, events, conditional probability, Bayes' theorem, independence, random variables, PMF, PDF, CDF, joint & marginal distributions, expectation, variance Foundation of ML models, Naive Bayes, probabilistic classifiers, uncertainty estimation.
Statistics Descriptive stats (mean, median, mode, std, IQR), Inferential stats, distributions (Normal, Binomial, Poisson), hypothesis testing, p-value, confidence intervals, correlation, regression coefficients Data analysis, A/B testing, understanding model outputs, EDA.
Calculus Derivatives, chain rule, partial derivatives, gradient, gradient descent intuitively, minima/maxima, cost function optimization Backpropagation in neural networks, optimization algorithms (SGD, Adam).
Resource Tip 3Blue1Brown's "Essence of Linear Algebra" and "Essence of Calculus" on YouTube are gold. StatQuest by Josh Starmer for statistics. These build deep intuition, not just formulas.

⛔ What to Avoid — From a 6-Year DS Practitioner
Tutorial Hell
Watching 100 tutorials without building anything. Always code along, always build.
Skipping Math
Using sklearn without understanding what's inside. Leads to cargo-cult ML — dangerous in production.
Tool Hopping
Switching frameworks every week. Go deep on PyTorch first before jumping to JAX or MXNet.
Skipping EDA
Jumping to models without understanding the data. 70% of DS work is data, not models.
Ignoring Business Context
A perfect model that doesn't solve the business problem is worthless.
No Version Control
Not using Git from day 1. Every project, every experiment must be in Git.
