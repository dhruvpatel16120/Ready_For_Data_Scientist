# 🤝 Contributing to Zero to Data Scientist

Thank you for your interest in contributing! This repository thrives on community collaboration. Whether you're a beginner or an expert, your contributions make a difference.

---

## 📋 How to Contribute

### 1. Fork & Clone
```bash
git fork https://github.com/dhruvpatel16120/Ready_For_Data_Scientist.git
git clone https://github.com/YOUR_USERNAME/Learning_ML.git
cd Learning_ML
```

### 2. Create a Branch
```bash
git checkout -b feature/level-X-topic-name
```

### 3. Make Your Changes
Follow the content guidelines below.

### 4. Commit & Push
```bash
git add .
git commit -m "Add: Level X - Topic Name - Brief description"
git push origin feature/level-X-topic-name
```

### 5. Open a Pull Request
Go to the original repository and open a PR with a clear description of your changes.

## 📝 Content Guidelines

### Jupyter Notebooks
- **Clear title** at the top with a markdown cell
- **Table of contents** for longer notebooks
- **Explanations before code** — Don't just dump code, explain the *why*
- **Comments in code** — Especially for complex logic
- **Output cells** — Run all cells before committing so outputs are visible
- **Clean naming**: `01_topic_name.ipynb`, not `Untitled.ipynb`

### Python Scripts
- Follow **PEP 8** style guidelines
- Include **docstrings** for functions and classes
- Add a **main block** (`if __name__ == "__main__":`)
- Include a brief header comment explaining the script's purpose

### README Files
- Every subfolder should have a `README.md`
- Include: **What you'll learn**, **Prerequisites**, **Topics covered**, **Resources**
- Use emoji sparingly but effectively for visual scanning

### File Naming Convention
```
01_topic_name.ipynb       # Numbered for order
utils.py                  # Helper scripts
README.md                 # Folder documentation
data/                     # Datasets (small only, <5MB)
```

## 🎯 What to Contribute

| Type | Examples | Difficulty |
|---|---|:---:|
| 📝 Notes & Theory | Markdown explanations, concept summaries | Easy |
| 🐍 Code Examples | Python scripts demonstrating concepts | Easy |
| 📓 Jupyter Notebooks | Step-by-step tutorials with output | Medium |
| 🧪 Practice Problems | Exercises with solutions | Medium |
| 📊 Projects | End-to-end data science projects | Hard |
| 📖 Resource Lists | Curated links to articles, videos, courses | Easy |
| 🐛 Bug Fixes | Fix typos, broken links, code errors | Easy |

---

## 🏗️ Folder Structure Rules

1. **Place content in the correct level folder** based on the roadmap
2. **Use numbered prefixes** (`01_`, `02_`) for ordering
3. **Keep datasets small** — Use links to large datasets instead
4. **Don't commit** virtual environments, `.env` files, or IDE configs

---

## ⚠️ Things to Avoid

- ❌ Plagiarized content — Always give credit
- ❌ Massive datasets committed to Git — Use Git LFS or external links
- ❌ Code without explanation — This is a learning repo
- ❌ Breaking existing content — Test before submitting
- ❌ Unrelated content — Stay within the Data Science domain
