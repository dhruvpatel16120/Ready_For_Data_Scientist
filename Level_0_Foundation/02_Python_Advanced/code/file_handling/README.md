# 📂 Mastery: Python File Handling

Welcome to the **File Handling** module! This guide covers everything from the basics of reading and writing text files to advanced operations like binary serialization, XML parsing, and manual file pointer manipulation.

---

## 🏗️ Project Architecture

To demonstrate industry best practices, this module is organized into logical layers:

### 1️⃣ Operational Modules (`modules/`)
Core logic is encapsulated into standalone modules for better maintainability.
- **`reading.py`**: High-level abstractions for reading files.
- **`writing.py`**: Standardized writing and appending logic.
- **`pointer.py`**: Mastering `seek()`, `tell()`, and `flush()`.

### 2️⃣ Data Assets (`test_file/`)
All demonstration data files are isolated in this directory to keep the workspace clean.
- `.txt`, `.csv`, `.json`, `.xml`, `.bin` files used across the lessons.

### 3️⃣ Interactive Lessons
| Step | Lesson | Description |
| :--- | :--- | :--- |
| **0** | `0_Manual_File_Handling.py` | Refactored modular entry point. |
| **1** | `1_Text_Files.py` | Basic text I/O with the `with` statement. |
| **2** | `2_CSV_Files.py` | Handling tabular data (comma-separated). |
| **3** | `3_JSON_Files.py` | Serialization and Deserialization of JSON objects. |
| **4** | `4_XML_Files.py` | Parsing Hierarchical data with ElementTree. |
| **5** | `5_Binary_Files.py` | Object persistence using the `pickle` module. |
| **6** | `6_Pointer_Methods.py`| Precision control with `seek()` and `tell()`. |
| **7** | `7_Modes_DeepDive.py` | mastering `r`, `w`, `a`, `r+`, `w+`, `a+`, `x`. |

---

## 🚀 Key Concepts Covered

- **The `with` Statement**: Understanding Context Managers for automatic resource cleanup.
- **File Modes**: Choosing between overwrite (`w`), append (`a`), and exclusive create (`x`).
- **Buffers & Pointing**: How Python manages data in memory vs. disk (`flush()`).
- **Structured Data**: Integrating CSV, JSON, and XML into your analytical workflows.

---

## 🛠️ Getting Started

1. **Setup**: All required data files are auto-generated by the scripts in the `test_file/` folder.
2. **Execution**: Run the scripts in order to see how complex data flows through Python.
   ```powershell
   python 1_Text_Files.py
   python 2_CSV_Files.py
   # ...
   ```
